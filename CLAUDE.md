# プロジェクト概要

[Marp](https://marp.app/) を使用したスライド作成リポジトリ。

# プロジェクトドキュメント構成

## ドキュメントの役割

| ファイル            | 役割                                                           |
| ------------------- | -------------------------------------------------------------- |
| `docs/PLAN.md`      | 要件・アイデアの音声入力による自由記述                         |
| `docs/SPEC.md`      | 仕様の整理。AIとの壁打ちによる認識合わせ                       |
| `docs/TODO.md`      | タスク管理。コンテキストリセット後も作業を再開できる状態を維持 |
| `docs/KNOWLEDGE.md` | 学びとナレッジの蓄積。同じ問題を繰り返さないための記録         |

## ワークフロー

1. **PLAN.md** に音声入力でアイデアや要望を書き出す
2. **SPEC.md** に整理する（Claudeと壁打ちして認識齟齬をなくす）
3. **TODO.md** を作成してタスク管理する
4. 学びや気づきがあれば **KNOWLEDGE.md** に記述する
